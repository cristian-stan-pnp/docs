## PayRetailers integration

PayRetailers allows processing **pay-ins** through local payment methods in LATAM (PSE, EFECTY, Pago Fácil, Khipu, SPEI, PIX, Transfer and Card Colombia), as well as **pay-outs**. Payments can be made through **checkout** or **redirect**.

***

## Pay-ins with PayRetailers

### General requirements

#### Supported currencies

Currently PayRetailers only supports the following currencies:

* USD
* ARS
* COP
* PEN
* MXN
* BRL

#### Required fields (`extra_data`)

All payment methods require including the following information within `extra_data`:

* **profile**
* `first_name`
* `last_name`
* `email`
* `document_identification_number`
* `phone.number`, `phone.prefix` (if applicable)

* **billing\_address**
* `city`, `country`, `zip_code`, `address1`, `state_code`

Example:

```
"extra_data": {    "checkout": {        "uuid": "C42043F9-B77C-4918-BD43-86C526B4F24D"    },    "profile": {        "first_name": "John",        "last_name": "Doe",        "email": "johndoe@gmail.com",        "document_identification_number": "92309089",        "phone": {            "number": "02614235138",            "prefix": "54"        }    },    "billing_address": {        "city": "Bogota",        "country": "COL",        "address1": "Calle Mayor, 23",        "zip_code": "111611",        "state_code": ""    }}
```

#### Additional URL parameters

| Parameter | Description              | Examples                                |
| --------- | ------------------------ | --------------------------------------- |
| `apm`     | Payment method selection | `?apm=PSE`, `?apm=PIX`, `?apm=TRANSFER` |
| `lang`    | Flow language            | `?lang=es`, `?lang=en`, `?lang=pt`      |

Default language: `es` (Spanish).

#### Sandbox environment behavior

The PayRetailers **sandbox** environment may not fully reflect the real behavior of payment methods.

To simulate different final states in sandbox, you can use the following `amount` values:

| `amount`         | Status              | Detail           |
| ---------------- | ------------------- | ---------------- |
| `11111`          | Paid                | Payment accepted |
| `44444`          | Rejected            | Payment canceled |
| `55555`          | Rejected            | Payment expired  |
| `66666`          | Rejected            | Payment error    |
| Any other value  | Waiting for response | Payment pending  |

## Pay-outs with PayRetailers

PayRetailers allows **payouts** (fund transfers) in the following countries:

| Country  | Currency |
| -------- | -------- |
| Brazil   | BRL      |
| Mexico   | MXN      |
| Chile    | CLP      |
| Colombia | COP      |
| Peru     | PEN      |
| Ecuador  | USD      |

There are two payout methods:

| Method   | Description                        | Compatible countries |
| -------- | ---------------------------------- | -------------------- |
| Transfer | Traditional bank transfer          | All of the above     |
| PIX      | Instant transfer via PIX key       | Brazil only          |

***

### Required fields in `extra_data`

#### `profile` (always required):

* `first_name`
* `last_name`
* `email`
* `document_identification_number`
* `document_identification_type` *(optional; only for COL, ECU and PER. Possible values: `NATIONAL_IDENTITY_DOCUMENT`, `FOREIGN_IDENTIFICATION_DOCUMENT`, `FISCAL_IDENTIFICATION_CODE`, `VALID_PASSPORT`)*
* `phone.number` *(required only if `country` is BRA, PER or ECU)*

#### `billing_address`:

* `address1`
* `country`

#### `payment` (depending on payout type):

**If the payout type is `Transfer`:**

* `bank_name` *(except Argentina)* – This field must contain the recipient bank's **code**. See the [Bank codes by country](#bank-codes-by-country) section for valid values per country.
* `account` [See specifications](#account-specifications-by-country-transfer)
* `account_type` *(except Argentina and Mexico)*
* `account_agency_number` *(Brazil only, required)* – Numeric code of the bank branch, provided by the bank. **Use only the numbers before the hyphen** (e.g.: `"1234-5"` → send `"1234"`).
* `aba_swift` *(optional, Brazil only)*
* `payout_beneficiary_type_code` *(optional, Chile only. Values: `PERSON` (default), `COMPANY`)*

**If the payout type is `PIX` (Brazil only):**

* `recipient_pix_key` *(optional. If not specified, the CPF is used as the PIX key)*

***

### `document_identification_type` values by country

The `document_identification_type` field is included within `profile`, is **optional** and **can only be used for Colombia, Ecuador and Peru**. If not specified, `NATIONAL_IDENTITY_DOCUMENT` is used by default. For other countries, this field should not be sent.

| Country | `document_identification_type` value | Equivalent document type     |
| ------- | ------------------------------------ | ---------------------------- |
| ARG     |                                      | DNI                          |
| BR      |                                      | CPF                          |
| CL      |                                      | RUT                          |
| MX      |                                      | CURP                         |
| COL     | `NATIONAL_IDENTITY_DOCUMENT`         | Citizenship card             |
| COL     | `FOREIGN_IDENTIFICATION_DOCUMENT`    | Foreign document             |
| ECU     | `NATIONAL_IDENTITY_DOCUMENT`         | DNI                          |
| ECU     | `FISCAL_IDENTIFICATION_CODE`         | RUC                          |
| ECU     | `VALID_PASSPORT`                     | Passport                     |
| PER     | `NATIONAL_IDENTITY_DOCUMENT`         | DNI                          |
| PER     | `FOREIGN_IDENTIFICATION_DOCUMENT`    | Foreign residency card       |
| PER     | `FISCAL_IDENTIFICATION_CODE`         | RUC                          |
| PER     | `VALID_PASSPORT`                     | Passport                     |

### Account specifications by country (Transfer)

#### Expected length / format

> **Data validation:** `^[0-9]+$` (only digits are allowed in `account`)

| Country        | Expected length    | Additional comments                                                                         |
| -------------- | ------------------ | ------------------------------------------------------------------------------------------- |
| AR (Argentina) | 22                 | CBU or CVU                                                                                  |
| BR (Brazil)    | 13                 | —                                                                                           |
| CL (Chile)     | 16                 | —                                                                                           |
| CO (Colombia)  | 16                 | —                                                                                           |
| MX (Mexico)    | 16, 18             | General (ACCOUNT\_CLABE): 18 digits Debit card number (CARD\_NUMBER): 16 digits             |
| EC (Ecuador)   | 12                 | —                                                                                           |
| PE (Peru)      | 10, 13, 14, 18, 20 | Depends on the bank. [See details](#peru-account-details)                                   |

##### Peru account details

| Bank / Institution       | Expected length | Comments / Examples                                      |
| ------------------------ | --------------- | -------------------------------------------------------- |
| Banco de Crédito (BCP)   | 13 / 14         | 13 = checking/master. 14 = savings                       |
| Interbank                | 13              | —                                                        |
| Banco Continental (BBVA) | 18 / 20         | E.g.: 18: `001106660100012345` 20: `00110666010001234512` |
| Scotiabank               | 10              | 3 branch digits + 7 account. E.g.: `0037651234`          |
| Other banks / FIs        | 20              | CCI required. E.g.: `00219400254640654321`               |

### PIX key format (`recipient_pix_key`)

| Key type | Regex format                                                   | Example                                |
| -------- | -------------------------------------------------------------- | -------------------------------------- |
| CPF      | `^[0-9]{11}$`                                                  | `12345678901`                          |
| Phone    | `^\+[1-9][0-9]\d{1,14}$`                                        | `+5510998765432`                       |
| Email    | `^[a-z0-9.!#$&'*+/=?^_{}~-]+@[a-z0-9.-]+$`                     | `pix@bcb.gov.br`                       |
| Random   | `[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}` | `123e4567-e89b-12d3-a456-426655440000` |

> Note: If the `recipient_pix_key` field is not explicitly provided within `extra_data.payment`, the value of the `document_identification_number` field will automatically be used as the PIX key, as long as it represents a valid CPF, simplifying integration in the most common cases.

### Sandbox environment testing

In the sandbox environment, you can simulate different payout results using the `amount` field value that you specify when [**generating the payment order**](/reference/#operation/generate-payment-order):

* **Any value other than `666`** simulates a **successful payout**.
* The value **`666`** simulates a **rejected payout**.
