## Introduction

PayPal is a payment service that allows you to process payments using the Paylands checkout or through a redirect.

## Restrictions and requirements

Currently only the EUR currency is supported.

When generating the payment order, at least the *profile* and *billing\_address* fields must be sent in order to reconcile PayPal payments.

```
"extra_data": {    "checkout": {        "uuid": "C42043F9-B77C-4918-BD43-86C526B4F24D"    },    "profile": {        "first_name": "John",        "last_name": "Doe",        "email": "johndoe@gmail.com",        "phone": {            "number": "666666666",            "prefix": "34"        }    },    "billing_address": {        "city": "Castellon",        "country": "ESP",        "address1": "Avinguda del Mar 23",        "zip_code": "12000",        "state_code": "CT"    }}
```

## STC (Set Transaction Context)")

Sending STC data allows you to enrich the transaction information with additional buyer and service details.

In the request used to [**generate the payment order**](/reference/#operation/generate-payment-order):

* Buyer data is sent in `extra_data.profile` and `extra_data.billing_address`
* OTA (Online Travel Agency) context data, related to the booking or travel service, is sent in `extra_data.payment`

OTA data is only required for certain industries, such as hotels.

### OTA - Hotels (MCC 7011)")

The following fields correspond to the OTA context for hotels.

All OTA fields must be sent within **`extra_data.payment`**.

| Field                    | Type                         | Description                                              | Example                         |
| ------------------------ | ---------------------------- | -------------------------------------------------------- | ------------------------------- |
| `buyer_creation_date`    | `string (ISO 8601 datetime)` | User registration date in the merchant's system          | `2012-12-09T19:14:55.277+00:00` |
| `ota_type`               | `string`                     | OTA industry type. For hotels it must be `"hotel"`       | `"hotel"`                       |
| `ota_service_start_date` | `string (ISO 8601 datetime)` | Stay start date (check-in)                               | `2026-06-01T15:00:00+00:00`     |
| `ota_service_end_date`   | `string (ISO 8601 datetime)` | Stay end date (check-out)                                | `2026-06-05T12:00:00+00:00`     |
| `ota_change_guest`       | `boolean`                    | Indicates whether the booking holder can be changed      | `false`                         |
| `ota_start_country`      | `string (ISO Alpha-2)`       | Country code where the stay begins                       | `"ES"`                          |
| `ota_start_city`         | `string`                     | City where the stay begins                               | `"Madrid"`                      |
| `ota_start_zipcode`      | `string`                     | Postal code where the stay begins                        | `"28080"`                       |

# PayLater amount restrictions

PayLater is supported in several countries, each with specific conditions based on amount and payment installments:

* Spain, Italy and the United Kingdom: amount between 30 and 2,000 euros (EUR), paid in 3 installments.
* Germany: amount between 99 and 5,000 euros (EUR), payment can be in 3, 6, 12, or 24 installments. They can also choose to pay within 30 days of purchase if the amount is between 1 and 1,000 euros (EUR).
* France and Australia: amount between 30 and 2,000 euros (EUR), paid in 4 installments.
* United Kingdom: if the amount is greater than 99 euros (EUR), 0% interest can be applied to the purchase.
* United States: amount between 30 and 1,500 euros (EUR), paid in 4 installments with the collection date every two weeks. There is an option to pay monthly, but the installments must be 6, 12, or 24 and the amount must be between 199 and 10,000 euros (EUR).
