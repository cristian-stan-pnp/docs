Two payment methods are supported: Card and [Bizum](/docs/apm/apm-bizum). Refund, confirmation, cancellation, and status retrieval operations are compatible with both payment methods. The remaining operations are oriented towards cards only.

## Restrictions and requirements

Redsys supports at least the following currencies: AUD, BGN, CAD, CHF, COP, CZK, DKK, EUR, GBP, HRK, HUF, ILS, INR, JPY, MYR, NOK, NZD, PEN, PLN, RUB, SAR, SEK, SGD, THB, USD, ZAR.

### Extra data

It is not required to specify `extra_data` when creating the order to carry out payments.

### Dynamic descriptor

The `dynamic_descriptor` field is optional and, if specified, can only contain a maximum of 38 characters and allows numbers, letters, and spaces.

### Reference

The `reference` field is optional. If specified, this value will appear in Redsys **Canales** within the `descripcion` field, along with the transaction `uuid` identifier.

The format displayed in Canales will be:

```
uuid_transaccion/reference
```

⚠️ Important: the merchant **must not include** the `uuid` or the slash `/` in the `reference` value. These are added automatically. The `uuid` corresponds to the transaction generated in our system.

For example, if the merchant sends as `reference`:

```
1234
```

In Canales it will be displayed as:

```
10FF94B5-FE7C-41A0-B956-BBBBAA6BCA77/1234
```

This field allows merchants to add their own information to facilitate operation traceability. The maximum size of this field is **88 characters**. If this limit is exceeded, the `reference` value will be **truncated** in Canales.
