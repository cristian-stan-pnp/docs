## Anti-fraud

Paylands offers a powerful anti-fraud system that allows you to redirect suspicious transactions to 3DS or reject them directly, without the payment being processed.

To do this, there are a series of available checks that can be grouped to form anti-fraud rules, each with its own score. The sum of the scores of all triggered rules will determine the score for the current operation.

### Anti-fraud configuration and creation

First, you need to go to the Security -> Anti Fraud tab in the panel. Here you will see a list of all available anti-fraud configurations and the number of associated services.

In the `Total Services` column, you can click on the number to see the services associated with the anti-fraud configuration. Remember that you need to associate the services that will use this anti-fraud configuration. A service can only be associated with one anti-fraud configuration.

![Anti fraud list](/images/docs/assets/images/antifraud_list-28332c37709f7d7da5e1caf87c100030.png)

To assign a service, go to Administration -> Service List -> Listing. Access the details of the service you want to associate with an Anti Fraud configuration. In the Anti Fraud tab, choose the desired one and update.

![Anti fraud service](/images/docs/assets/images/associate_antifraud_service-af03a69630483f7eaa9c50e4f4e49680.png)

## Security lists

Within the security lists, you can configure elements (IP, BIN, UUID, EXTERNAL\_ID and TOKEN) to be marked as safe, risky, or blocked.

To access each security list, simply go to **Security** and click on the desired list.

In this section, you can see each available list where you can enable or disable them and check the services associated with them. You can even unlink the services you want from that list.

![ColorList table](/images/docs/assets/images/colorlist_table-ec48062c05b366a4785978833e314c1e.png)

![ColorList services](/images/docs/assets/images/colorlist_services-5d2d074e0b68dd664ee3e6f76f2608db.png)

In the detail of each list, you can create, modify and delete elements. Remember that you will also need to associate as many services as you want to use this list.

![ColorList detail](/images/docs/assets/images/colorlist_detail-ed1a5fc228afbaf082d9de64dc48f594.png)

![ColorList add service](/images/docs/assets/images/colorlist_add_service-77491fc9efcb98ef490ee2cae1f8299f.png)

### Whitelists

From this section, you can configure whitelists that can be associated with existing services.

These lists allow you to configure checks to prevent a trusted operation from entering risk or fraud evaluation. The parameters that can be checked to validate that the operation is trusted are the following:

* IP: IP address from which the operation is performed.
* BIN: First 6 digits of the card number.
* UUID: Card identifier.
* EXTERNAL\_ID: Customer identifier.
* TOKEN: Card identifier token.

### Blacklists

From this section, you can configure blacklists that can be associated with existing services.

These lists allow you to configure checks to prevent a suspicious operation from being processed. The parameters that can be checked to validate that the operation should be blocked are the following:

* IP: IP address from which the operation is performed.
* BIN: First 6 digits of the card number.
* UUID: Card identifier.
* EXTERNAL\_ID: Customer identifier.
* TOKEN: Card identifier token.

### Grey lists

From this section, you can configure grey lists that can be associated with existing services.

These lists allow you to configure checks to force a suspicious operation to be processed as a secure order (3DS). The parameters that can be checked to validate that the operation's security level should be increased are the following:

* IP: IP address from which the operation is performed.
* BIN: First 6 digits of the card number.
* UUID: Card identifier.
* EXTERNAL\_ID: Customer identifier.
* TOKEN: Card identifier token.
