# Klap

## Redirect integration

To integrate via redirect, the user must be present at the time of payment, which must be processed through a browser. First, you need to [generate a payment order](/reference/#operation/generate-payment-order), specifying the email.

```
  "extra_data": {    "profile": {      "first_name": " ",      "last_name": " ",      "email": "test@gmail.com"    }  }
```

As the service identifier, you must specify the **service\_uuid** of the FLOW service that you have activated in your **Paylands Backend → Manage → Payment services → [List](https://backend.paylands.com/payment-services/list)**.

Once the order is generated, the user must be redirected to [the payment screen](/reference/#operation/redirection-payment) specifying the order token and the APM name in the query:

Build the redirect URL by replacing the **token** with the value returned in the previous call.

```
https://api.paylands.com/v1/sandbox/payment/process/{token}?apm=KLAP
```

Once the user is redirected to this screen, a coupon will be displayed with which the user can make the payment at a branch. When the payment is completed, they will be redirected back to the OK or KO URL, depending on the payment result.

## Testing

For more information on how to test this payment method with each of its providers, visit the FLOW documentation.

## Custom fields

Optionally, within the `extra_data` object you can include the `payment_reason` field, which allows you to specify a payment description.

```
  "extra_data": {    "payment": {      "payment_reason": "Payment description"    }  }
```
