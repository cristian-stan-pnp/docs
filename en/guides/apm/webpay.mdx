# Webpay

## Redirect integration with PAYRETAILERS[​](#redirect-integration-with-payretailers "Direct link to Redirect integration with PAYRETAILERS")

This method requires the user to be present at the time of payment, which must be processed through a browser. First, you need to [generate a payment order](/reference/#operation/generate-payment-order) specifying the following user fields to avoid having to fill them in at a later step.

```
  "extra_data": {      "profile": {          "first_name": "John",          "last_name": "Doe",          "email": "johndoe@gmail.com",          "document_identification_number": "92309089",          "phone": {              "number": "02614235138",              "prefix": "54"          }      },      "billing_address": {          "city": "Castellon",          "country": "CHL",          "address1": "Avinguda del Mar 23",          "zip_code": "12000",          "state_code": "CT"      }  }
```

## Redirect integration with FLOW[​](#redirect-integration-with-flow "Direct link to Redirect integration with FLOW")

To integrate via redirect, the user must be present at the time of payment, which must be processed through a browser. First, you need to [generate a payment order](/reference/#operation/generate-payment-order), specifying the email.

```
  "extra_data": {    "profile": {      "first_name": " ",      "last_name": " ",      "email": "test@gmail.com"    }  }
```

As the service identifier, you must specify the **service\_uuid** of the PAYRETAILERS or FLOW service that you have activated in your **Paylands Backend → Manage → Payment services → [List](https://backend.paylands.com/payment-services/list)**.

Once the order is generated, the user must be redirected to [the payment screen](/reference/#operation/redirection-payment) specifying the order token and the APM name in the query:

Build the redirect URL by replacing the **token** with the value returned in the previous call.

```
https://api.paylands.com/v1/sandbox/payment/process/{token}?apm=WEBPAY
```

Once the user is redirected to this screen, they can enter their card details to complete the payment. When the payment is completed, they will be redirected back to the OK or KO URL, depending on the payment result.

## Testing[​](#testing "Direct link to Testing")

For more information on how to test this payment method with each of its providers, visit the PAYRETAILERS or FLOW documentation.

## Custom fields[​](#custom-fields "Direct link to Custom fields")

Optionally, when using WEBPAY with FLOW, within the `extra_data` object you can include the `payment_reason` field, which allows you to specify a payment description.

```
  "extra_data": {    "payment": {      "payment_reason": "Payment description"    }  }
```
