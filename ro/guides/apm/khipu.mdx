# Khipu

## Integrare prin redirecționare pentru FLOW

Pentru a-l integra prin redirecționare, utilizatorul trebuie să fie prezent în momentul plății, care trebuie procesată printr-un browser. În primul rând, trebuie să [generăm o comandă de plată](/reference/#operation/generate-payment-order), indicând email-ul.

```
  "extra_data": {    "profile": {      "first_name": " ",      "last_name": " ",      "email": "test@gmail.com"    }  }
```

Ca identificator de serviciu, trebuie indicat **service\_uuid** al serviciului FLOW pe care îl aveți activat în **Backend-ul Paylands → Administrare → Servicii de plată → [Listă](https://backend.paylands.com/payment-services/list)**.

Odată generată comanda, utilizatorul trebuie redirecționat către [ecranul de plată](/reference/#operation/redirection-payment) indicând token-ul comenzii și în query, numele APM-ului:

Construim URL-ul de redirecționare înlocuind **token**-ul cu valoarea returnată în apelul anterior.

```
https://api.paylands.com/v1/sandbox/payment/process/{token}?apm=KHIPU
```

Odată ce utilizatorul este redirecționat către acest ecran, va putea selecta banca sa pentru a autoriza tranzacția. Când finalizează plata, va fi redirecționat înapoi la URL-ul OK sau KO, în funcție de rezultatul plății.

## Integrare prin redirecționare pentru Payretailers

Pentru a-l integra prin redirecționare, utilizatorul trebuie să fie prezent în momentul plății, care trebuie procesată printr-un browser. În primul rând, trebuie să [generăm o comandă de plată](/reference/#operation/generate-payment-order), indicând first\_name, last\_name, email, document\_identification\_number, phone, city, country, address1, zip\_code, state\_code.

```
  "extra_data": {    "checkout": {        "uuid": "C42043F9-B77C-4918-BD43-86C526B4F24D"    },    "profile": {        "first_name": "Kobe",        "last_name": "Bryant",        "email": "andreu.sanchez@paynopain.com",        "document_identification_number": "92309089",        "phone": {            "number": "02614235138",            "prefix": "54"        }    },    "billing_address": {        "city": "Castellon",        "country": "COL",        "address1": "Avinguda del Mar 23",        "zip_code": "12000",        "state_code": "CT"    }  }
```

Ca identificator de serviciu, trebuie indicat **service\_uuid** al serviciului Payretailers pe care îl aveți activat în **Backend-ul Paylands → Administrare → Servicii de plată → [Listă](https://backend.paylands.com/payment-services/list)**.

Odată generată comanda, utilizatorul trebuie redirecționat către [ecranul de plată](/reference/#operation/redirection-payment) indicând token-ul comenzii și în query, numele APM-ului:

Construim URL-ul de redirecționare înlocuind **token**-ul cu valoarea returnată în apelul anterior.

```
https://api.paylands.com/v1/sandbox/payment/process/{token}?apm=KHIPU
```

## Testare

Pentru mai multe informații despre cum să testați această metodă de plată la fiecare dintre furnizorii săi, vizitați documentația FLOW și Payretailers.

## Câmpuri personalizate

Opțional, în cadrul obiectului `extra_data` puteți include câmpul `payment_reason`, care vă permite să indicați o descriere a plății.

```
  "extra_data": {    "payment": {      "payment_reason": "Payment description"    }  }
```
