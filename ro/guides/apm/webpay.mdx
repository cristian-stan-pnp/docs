# Webpay

## Integrare prin redirecționare cu PAYRETAILERS

Această metodă necesită ca utilizatorul să fie prezent în momentul plății, care trebuie procesată printr-un browser. În primul rând, trebuie să [generăm o comandă de plată](/reference/#operation/generate-payment-order) indicând următoarele câmpuri despre utilizator pentru a evita ca utilizatorul să fie nevoit să le completeze într-un pas ulterior.

```
  "extra_data": {      "profile": {          "first_name": "John",          "last_name": "Doe",          "email": "johndoe@gmail.com",          "document_identification_number": "92309089",          "phone": {              "number": "02614235138",              "prefix": "54"          }      },      "billing_address": {          "city": "Castellon",          "country": "CHL",          "address1": "Avinguda del Mar 23",          "zip_code": "12000",          "state_code": "CT"      }  }
```

## Integrare prin redirecționare cu FLOW

Pentru a-l integra prin redirecționare, utilizatorul trebuie să fie prezent în momentul plății, care trebuie procesată printr-un browser. În primul rând, trebuie să [generăm o comandă de plată](/reference/#operation/generate-payment-order), indicând email-ul.

```
  "extra_data": {    "profile": {      "first_name": " ",      "last_name": " ",      "email": "test@gmail.com"    }  }
```

Ca identificator de serviciu, trebuie indicat **service\_uuid** al serviciului PAYRETAILERS sau FLOW pe care îl aveți activat în **Backend-ul Paylands → Administrare → Servicii de plată → [Listă](https://backend.paylands.com/payment-services/list)**.

Odată generată comanda, utilizatorul trebuie redirecționat către [ecranul de plată](/reference/#operation/redirection-payment) indicând token-ul comenzii și în query, numele APM-ului:

Construim URL-ul de redirecționare înlocuind **token**-ul cu valoarea returnată în apelul anterior.

```
https://api.paylands.com/v1/sandbox/payment/process/{token}?apm=WEBPAY
```

Odată ce utilizatorul este redirecționat către acest ecran, va putea introduce datele cardului pentru a finaliza plata. Când finalizează plata, va fi redirecționat înapoi la URL-ul OK sau KO, în funcție de rezultatul plății.

## Testare

Pentru mai multe informații despre cum să testați această metodă de plată la fiecare dintre furnizorii săi, vizitați documentația PAYRETAILERS sau FLOW.

## Câmpuri personalizate

Opțional, în cazul în care utilizați WEBPAY cu FLOW, în cadrul obiectului `extra_data` puteți include câmpul `payment_reason`, care vă permite să indicați o descriere a plății.

```
  "extra_data": {    "payment": {      "payment_reason": "Payment description"    }  }
```
