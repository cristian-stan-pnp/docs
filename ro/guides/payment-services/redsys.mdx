Sunt acceptate două metode de plată: Card și [Bizum](/docs/apm/apm-bizum). Operațiunile de rambursare, confirmare, anulare și recuperare a stării sunt compatibile cu ambele metode de plată. Restul operațiunilor sunt destinate exclusiv cardurilor.

## Restricții și cerințe

Redsys acceptă cel puțin următoarele monede: AUD, BGN, CAD, CHF, COP, CZK, DKK, EUR, GBP, HRK, HUF, ILS, INR, JPY, MYR, NOK, NZD, PEN, PLN, RUB, SAR, SEK, SGD, THB, USD, ZAR.

### Extra data

Nu este necesar să se indice `extra_data` la crearea comenzii pentru efectuarea plăților.

### Dynamic descriptor

Câmpul `dynamic_descriptor` este opțional și, în cazul în care este indicat, poate conține maximum 38 de caractere și permite numere, litere și spații.

### Reference

Câmpul `reference` este opțional. În cazul în care este indicat, această valoare va apărea în **Canales** din Redsys în câmpul `descripcion`, împreună cu identificatorul `uuid` al tranzacției.

Formatul afișat în Canales va fi:

```
uuid_transaccion/reference
```

⚠️ Important: comerciantul **nu trebuie să includă** `uuid`-ul și nici bara `/` în valoarea `reference`. Acestea se adaugă automat. `uuid`-ul corespunde tranzacției generate în sistemul nostru.

De exemplu, dacă comerciantul trimite ca `reference`:

```
1234
```

În Canales se va afișa:

```
10FF94B5-FE7C-41A0-B956-BBBBAA6BCA77/1234
```

Acest câmp permite comercianților să adauge informații proprii pentru a facilita trasabilitatea operațiunilor. Dimensiunea maximă a acestui câmp este de **88 de caractere**. Dacă se depășește această limită, valoarea `reference` va fi **trunchiată** în Canales.
