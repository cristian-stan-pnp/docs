## Integrarea PayRetailers

PayRetailers permite procesarea **pay-in-urilor** prin metode de plată locale în LATAM (PSE, EFECTY, Pago Fácil, Khipu, SPEI, PIX, Transfer și Tarjeta Colombia), precum și **pay-out-uri**. Plățile pot fi efectuate prin **checkout** sau **redirecționare**.

***

## Payins cu Payretailers

### Cerințe generale

#### Monede acceptate

În prezent, PayRetailers acceptă doar următoarele monede:

* USD
* ARS
* COP
* PEN
* MXN
* BRL

#### Date obligatorii (`extra_data`)

Toate metodele de plată necesită includerea următoarelor informații în `extra_data`:

* **profile**
* `first_name`
* `last_name`
* `email`
* `document_identification_number`
* `phone.number`, `phone.prefix` (dacă este cazul)

* **billing\_address**
* `city`, `country`, `zip_code`, `address1`, `state_code`

Exemplu:

```
"extra_data": {    "checkout": {        "uuid": "C42043F9-B77C-4918-BD43-86C526B4F24D"    },    "profile": {        "first_name": "John",        "last_name": "Doe",        "email": "johndoe@gmail.com",        "document_identification_number": "92309089",        "phone": {            "number": "02614235138",            "prefix": "54"        }    },    "billing_address": {        "city": "Bogota",        "country": "COL",        "address1": "Calle Mayor, 23",        "zip_code": "111611",        "state_code": ""    }}
```

#### Parametri suplimentari în URL

| Parametru | Descriere                      | Exemple                                  |
| --------- | ------------------------------ | ---------------------------------------- |
| `apm`     | Selectarea metodei de plată    | `?apm=PSE`, `?apm=PIX`, `?apm=TRANSFER` |
| `lang`    | Limba fluxului                 | `?lang=es`, `?lang=en`, `?lang=pt`       |

Limba implicită: `es` (spaniolă).

#### Comportamentul mediului sandbox

Mediul **sandbox** al PayRetailers poate să nu reflecte complet comportamentul real al metodelor de plată.

Pentru a simula diferite stări finale în sandbox, se pot utiliza următoarele valori ale `amount`:

| `amount`              | Stare               | Detaliu              |
| --------------------- | ------------------- | -------------------- |
| `11111`               | Plătit              | Plată acceptată      |
| `44444`               | Respins             | Plată anulată        |
| `55555`               | Respins             | Plată expirată       |
| `66666`               | Respins             | Eroare la plată      |
| Orice altă valoare    | Așteptare răspuns   | Plată în așteptare   |

### Metode de plată

#### PSE (Columbia)

* **Țară:** `COL`
* **Monede:** COP, USD
* **APM:** PSE


* **Țară:** `ECU`, `BRA`
* **Monede:** USD, BRL
* **APM:** TRANSFER

##### Bănci acceptate (`payment.bank_name`)

| Țară    | Valori permise                       |
| ------- | ------------------------------------ |

* Ecuador → `BANCO_PICHINCHA`

## Payouts cu Payretailers

Payretailers permite efectuarea de **payouts** (transferuri de fonduri) în următoarele țări:

| Țară     | Monedă |
| -------- | ------ |
| Brazilia | BRL    |
| Mexic    | MXN    |
| Chile    | CLP    |
| Columbia | COP    |
| Peru     | PEN    |
| Ecuador  | USD    |

Există două metode de payout:

| Metodă   | Descriere                                  | Țări compatibile     |
| -------- | ------------------------------------------ | -------------------- |
| Transfer | Transfer bancar tradițional                | Toate cele de mai sus |
| PIX      | Transfer imediat prin cheie PIX            | Doar Brazilia        |

***

### Câmpuri obligatorii în `extra_data`

#### `profile` (întotdeauna necesar):

* `first_name`
* `last_name`
* `email`
* `document_identification_number`
* `document_identification_type` *(opțional; doar pentru COL, ECU și PER. Valori posibile: `NATIONAL_IDENTITY_DOCUMENT`, `FOREIGN_IDENTIFICATION_DOCUMENT`, `FISCAL_IDENTIFICATION_CODE`, `VALID_PASSPORT`)*
* `phone.number` *(necesar doar dacă `country` este BRA, PER sau ECU)*

#### `billing_address`:

* `address1`
* `country`

#### `payment` (în funcție de tipul de payout):

**Dacă tipul de payout este `Transfer`:**

* `bank_name` *(cu excepția Argentinei)* – Acest câmp trebuie să conțină **codul** băncii destinatarului. Consultă secțiunea [Coduri de bănci pe țară](#coduri-de-bănci-pe-țară) pentru a vedea valorile valide în funcție de țară.
* `account` [Vezi specificațiile](#specificații-pentru-account-pe-țară-transfer)
* `account_type` *(cu excepția Argentinei și Mexicului)*
* `account_agency_number` *(doar Brazilia, obligatoriu)* – Cod numeric al agenției bancare, furnizat de bancă. **Se folosesc doar cifrele de dinaintea cratimei** (ex: `"1234-5"` → se trimite `"1234"`).
* `aba_swift` *(opțional și doar Brazilia)*
* `payout_beneficiary_type_code` *(opțional, doar Chile. Valori: `PERSON` (implicit), `COMPANY`)*

**Dacă tipul de payout este `PIX` (doar Brazilia):**

* `recipient_pix_key` *(opțional. Dacă nu este indicat, se folosește CPF-ul ca cheie PIX)*

***

### Valorile `document_identification_type` pe țară

Câmpul `document_identification_type` se include în `profile`, este **opțional** și **poate fi folosit doar pentru țările Columbia, Ecuador și Peru**. Dacă nu este indicat, se va utiliza implicit `NATIONAL_IDENTITY_DOCUMENT`. Pentru restul țărilor, acest câmp nu trebuie trimis.

| Țară | Valoare `document_identification_type` | Tip de document echivalent      |
| ---- | -------------------------------------- | ------------------------------- |
| ARG  |                                        | DNI                             |
| BR   |                                        | CPF                             |
| CL   |                                        | RUT                             |
| MX   |                                        | CURP                            |
| COL  | `NATIONAL_IDENTITY_DOCUMENT`           | Carte de identitate             |
| COL  | `FOREIGN_IDENTIFICATION_DOCUMENT`      | Document de străin              |
| ECU  | `NATIONAL_IDENTITY_DOCUMENT`           | DNI                             |
| ECU  | `FISCAL_IDENTIFICATION_CODE`           | RUC                             |
| ECU  | `VALID_PASSPORT`                       | Pașaport                        |
| PER  | `NATIONAL_IDENTITY_DOCUMENT`           | DNI                             |
| PER  | `FOREIGN_IDENTIFICATION_DOCUMENT`      | Carnet de rezidență             |
| PER  | `FISCAL_IDENTIFICATION_CODE`           | RUC                             |
| PER  | `VALID_PASSPORT`                       | Pașaport                        |

***

### Valorile `account_type` pe țară (Transfer)

| account\_type  | BR | CO | CL | PE | EC | MX |
| -------------- | -- | -- | -- | -- | -- | -- |
| CHECKING       | ✓  | ✓  | ✓  | ✓  | ✓  |    |
| SAVINGS        | ✓  | ✓  | ✓  | ✓  | ✓  |    |
| RUT            |    |    | ✓  |    |    |    |
| ACCOUNT\_CLABE |    |    |    |    |    | ✓  |
| CARD\_NUMBER   |    |    |    |    |    | ✓  |

> Argentina și Mexic nu necesită indicarea `account_type`, se aplică o valoare implicită internă:
>
> * Argentina: CBU sau CVU
> * Mexic: tip General (ACCOUNT\_CLABE)

### Specificații pentru `account` pe țară (Transfer)

#### Lungime așteptată / format

> **Validarea datelor:** `^[0-9]+$` (doar cifre sunt permise în `account`)

| Țară           | Lungime așteptată  | Comentarii suplimentare                                                                     |
| -------------- | ------------------ | ------------------------------------------------------------------------------------------- |
| AR (Argentina) | 22                 | CBU sau CVU                                                                                 |
| BR (Brazilia)  | 13                 | —                                                                                           |
| CL (Chile)     | 16                 | —                                                                                           |
| CO (Columbia)  | 16                 | —                                                                                           |
| MX (Mexic)     | 16, 18             | General (ACCOUNT\_CLABE): 18 cifre Număr de card de debit (CARD\_NUMBER): 16 cifre          |
| EC (Ecuador)   | 12                 | —                                                                                           |
| PE (Peru)      | 10, 13, 14, 18, 20 | Depinde de bancă. [Vezi detalii](#detalii-pentru-peru-account)                           |

##### Detalii pentru Peru (`account`)

| Bancă / Instituție       | Lungime așteptată | Comentarii / Exemple                                             |
| ------------------------ | ----------------- | ---------------------------------------------------------------- |
| Banco de Crédito (BCP)   | 13 / 14           | 13 = cont curent/maestru. 14 = economii                         |
| Interbank                | 13                | —                                                                |
| Banco Continental (BBVA) | 18 / 20           | Ex.: 18: `001106660100012345` 20: `00110666010001234512`         |
| Scotiabank               | 10                | 3 cifre agenție + 7 cont. Ex.: `0037651234`                     |
| Alte bănci / IFI-uri     | 20                | CCI obligatoriu. Ex.: `00219400254640654321`                     |

### Formatul cheii PIX (`recipient_pix_key`)

| Tip cheie  | Format regex                                                   | Exemplu                                |
| ---------- | -------------------------------------------------------------- | -------------------------------------- |
| CPF        | `^[0-9]{11}$`                                                  | `12345678901`                          |
| Telefon    | `^\+[1-9][0-9]\d{1,14}$`                                        | `+5510998765432`                       |
| Email      | `^[a-z0-9.!#$&'*+/=?^_{}~-]+@[a-z0-9.-]+$`                     | `pix@bcb.gov.br`                       |
| Aleatoriu  | `[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}` | `123e4567-e89b-12d3-a456-426655440000` |

> Notă: Dacă nu se indică explicit câmpul `recipient_pix_key` în `extra_data.payment`, se va utiliza automat valoarea câmpului `document_identification_number` ca cheie PIX, cu condiția ca aceasta să reprezinte un CPF valid, simplificând integrarea în cazurile cele mai frecvente.

### Teste în mediul sandbox

În mediul sandbox, poți simula diferite rezultate în operațiunile de payout utilizând valoarea câmpului `amount` pe care o vei indica la [**generarea ordinului de plată**](/reference/#operation/generate-payment-order):

* **Orice valoare diferită de `666`** simulează un **payout reușit**.
* Valoarea **`666`** simulează un **payout respins**.
