---
title: 'Verificarea Titularului'
description: 'Cum să verifici identitatea titularului unui card în Paylands'
---

Verificarea poate fi realizată prin numele titularului sau printr-un identificator legal al titularului, în funcție de metoda selectată.

***

## Metode de verificare

SHV permite verificarea titularității unui `source` utilizând una dintre următoarele metode:

* `NAME` (**implicit**): verifică dacă numele titularului indicat coincide cu titularul real al mijlocului de plată.
* `IDENTIFICATION_NUMBER`: verifică dacă identificatorul legal indicat corespunde titularului real al mijlocului de plată.

Metoda poate fi indicată prin câmpul `verification_method`. Dacă nu este specificată, se utilizează `NAME`.

***

## În ce fluxuri se poate utiliza?

SHV poate fi aplicat în **trei fluxuri**:

1. **[Endpoint specific de verificare](/reference/#operation/verify-source-holder)**

   * Permite **verificarea unui `source` față de un titular** și **obținerea rezultatului** verificării.
   * Returnează rezultatul de coincidență: `MATCH`, `NO_MATCH`, `PARTIAL_MATCH` (când verificarea este efectuată).

2. **Stocarea unui mijloc de plată ([tokenizare card](/reference/#operation/save-card) / [creare cont bancar](/reference/#operation/save-bank))**

   * În endpointul de **stocare** a cardului sau contului bancar, dacă se trimit câmpurile necesare, SHV validează dacă `source`-ul care se dorește a fi stocat **aparține titularului indicat**.
   * În funcție de politicile configurate, fluxul poate fi **blocat** și se poate evita stocarea `source`-ului.

3. **Flux de plată (payins și payouts)**

   * La [**generarea comenzii de plată**](/reference/#operation/generate-payment-order), se poate indica în `extra_data` corpul `source_holder_verification`.

   * Înainte de a încerca plata, se execută o verificare SHV a `source`-ului cu care se dorește efectuarea plății.

   * Permite **blocarea plății** dacă:

     * verificarea **nu poate fi efectuată** (conform politicii), sau
     * rezultatul **nu coincide** (conform `accepted_verification_results`).

***

## Endpoint specific: verificarea titularului unui source

### Exemplu de cerere cu NAME

```
{  "signature": "341f7de8e6fc49da8d8736473af6b03a",  "source_uuid": "EEE99130-218B-4E22-90E2-E55B845DF0D4",  "service": "E8952860-5441-45A2-9233-33651DABA250",  "holder": {    "first_name": "Luis",    "last_name": "Pérez López"  }}
```

### Exemplu de cerere cu IDENTIFICATION\_NUMBER

> Când `verification_method` este `IDENTIFICATION_NUMBER`, trebuie trimis `holder.identification_number`. Câmpul `first_name` rămâne necesar pentru verificare.

```
{  "signature": "341f7de8e6fc49da8d8736473af6b03a",  "source_uuid": "EEE99130-218B-4E22-90E2-E55B845DF0D4",  "service": "E8952860-5441-45A2-9233-33651DABA250",  "holder": {    "first_name": "Luis",    "last_name": "Pérez López",    "identification_number": "B88353412"  },  "verification_method": "IDENTIFICATION_NUMBER"}
```

### Exemplu de răspuns

```
{  "message": "OK",  "code": 200,  "current_time": "2026-02-24T14:35:01+0100",  "verification": {    "uuid": "A8CABF0C-C690-46C7-9F4B-E6AC3F929799",    "status": "PERFORMED",    "match_result": "MATCH",    "error": null,    "policy_action": null,    "verification_method": "NAME"  }}
```

## Aplicarea SHV la stocarea unui source sau la plăți

În fluxurile de **stocare a unui source** și **plată**, corpul trimis este același:

```
{  "source_holder_verification": {    "enabled": true,    "accepted_verification_results": ["MATCH", "PARTIAL_MATCH"],    "allow_unavailable": true,    "allow_error": false,    "first_name": "Pedro",    "last_name": "Perez"  }}
```

### Ce face acest corp?

* Dacă `enabled = true`, se încearcă executarea SHV în timpul fluxului.

* Permite definirea **politicilor**:

  * ce rezultate sunt acceptabile (`accepted_verification_results`),
  * ce să se facă dacă verificarea **nu este disponibilă** sau **nu se realizează** (`allow_unavailable`),
  * ce să se facă în cazul **erorilor** (`allow_error`).

* Permite informarea `first_name` și `last_name`. Dacă nu sunt trimise:

  * În **plăți**, se încearcă obținerea din `extra_data.profile`.
  * În **stocare**, se încearcă obținerea din `card_holder` / `account_holder`.

* Permite selectarea metodei de verificare prin `verification_method`.

Pentru ca verificarea să poată fi realizată, datele necesare trebuie să fie disponibile în operațiune:

* **Numele titularului** (`first_name`) trebuie să fie prezent, fie trimis în `source_holder_verification`, fie obținut din `extra_data.profile.first_name`.
* Dacă metoda selectată este `IDENTIFICATION_NUMBER`, trebuie să fie disponibil și **identificatorul legal al titularului** (`identification_number`), fie trimis explicit, fie obținut din `extra_data.profile.document_identification_number`.

## Câmpurile `source_holder_verification`

| Câmp                            | Tip                 | Obligatoriu | Default | Descriere                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ------------------------------- | ------------------- | ----------- | ------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `enabled`                       | `boolean`           | **Da**      | —       | Activează verificarea titularului mijlocului de plată. Când este setat pe `true`, platforma efectuează o verificare SHV înainte de a continua cu fluxul (de exemplu, înainte de executarea plății).                                                                                                                                                                                                                                               |
| `verification_method`           | `string` *(enum)*   | Nu          | `NAME`  | Metoda utilizată pentru verificarea titularității mijlocului de plată. Valori permise: `NAME` sau `IDENTIFICATION_NUMBER`. Dacă se setează `NAME`, verificarea se bazează pe compararea numelui titularului indicat cu titularul real al mijlocului de plată. Dacă se setează `IDENTIFICATION_NUMBER`, verificarea se bazează pe compararea identificatorului legal indicat cu titularul real al mijlocului de plată.                                |
| `accepted_verification_results` | `string[]` *(enum)* | Nu          | —       | Lista rezultatelor de verificare care sunt considerate acceptabile pentru a continua cu operațiunea. Dacă este omisă sau goală, fluxul continuă indiferent de rezultatul SHV. Valori permise: `MATCH`, `NO_MATCH`, `PARTIAL_MATCH`.                                                                                                                                                                                                              |
| `allow_unavailable`             | `boolean`           | Nu          | `true`  | Permite continuarea fluxului dacă verificarea **nu a fost realizată** deoarece nu este suportată de bancă sau de tipul/marca cardului, sau deoarece, deși ar putea fi suportată, banca nu o execută.                                                                                                                                                                                                                                             |
| `allow_error`                   | `boolean`           | Nu          | `false` | Determină comportamentul atunci când verificarea nu poate fi finalizată din cauza defecțiunilor/erorilor tranzitorii (de ex. timeout sau indisponibilitate temporară). Când este `true`, aceste erori **nu** blochează fluxul. Când este `false`, aceste erori **blochează** operațiunea.                                                                                                                                                          |
| `first_name`                    | `string`            | Nu          | —       | Prenumele titularului mijlocului de plată. Dacă nu este trimis: în plăți se încearcă obținerea din `extra_data.profile`; în stocare, din `card_holder` / `account_holder`. Această informație trebuie să fie disponibilă pentru ca verificarea să poată fi realizată.                                                                                                                                                                              |
| `last_name`                     | `string`            | Nu          | —       | Numele de familie al titularului mijlocului de plată (aceeași logică ca `first_name`).                                                                                                                                                                                                                                                                                                                                                           |
| `identification_number`         | `string`            | Condiționat | —       | Identificatorul legal al titularului mijlocului de plată (de exemplu, NIF, DNI, pașaport sau număr de înregistrare al companiei). Obligatoriu când `verification_method` este `IDENTIFICATION_NUMBER`. Dacă nu este trimis, în plăți se încearcă obținerea din `extra_data.profile.document_identification_number`.                                                                                                                                |

## Răspuns SHV (format comun)

Indiferent de endpoint/flux, blocul de răspuns SHV este returnat în această formă:

```
{  "source_holder_verification": {    "uuid": "ED17EDE3-51A7-47A9-BD13-50448587278C",    "status": "PERFORMED",    "match_result": "NO_MATCH",    "error": null,    "policy_action": "ALLOWED",    "verification_method": "NAME"  }}
```

### Semnificația câmpurilor

* `uuid`: identificator unic al verificării efectuate.
* `status`: starea finală a tentativei de verificare.
* `match_result`: rezultatul de coincidență (dacă este aplicabil).
* `error`: eroarea care s-a putut produce în timpul verificării.
* `policy_action`: se aplică doar fluxurilor de **creare source** și **plată** (când se trimite `source_holder_verification` cu politici). Indică decizia finală luată privind continuarea fluxului.

***

## Stări ale `status`

Valori posibile:

* `PERFORMED` Verificarea a putut fi realizată. Se așteaptă ca `match_result` să fie informat (`MATCH`, `PARTIAL_MATCH` sau `NO_MATCH`).

* `NOT_PERFORMED` Verificarea nu a fost realizată (din motive care nu sunt neapărat legate de suport).

* `NOT_SUPPORTED` Verificarea nu este suportată pentru banca / tipul de cont / marca sau tipul cardului.

* `FAILED` Eșec în timpul executării verificării.

* `ERROR` Eroare tranzitorie sau tehnică (de exemplu, timeout sau indisponibilitate temporară).

***

## Rezultate de coincidență `match_result`

O verificare SHV poate returna:

* `MATCH` — Datele titularului coincid complet cu cele ale titularului real al mijlocului de plată.
* `PARTIAL_MATCH` — Coincidență aproximativă cu mici discrepanțe.
* `NO_MATCH` — Datele titularului nu coincid cu titularul real.

***

## Acțiuni de politică `policy_action`

> Se aplică în fluxurile de **creare source** și **plată** când s-au trimis politici în `source_holder_verification`.

Valori posibile:

* `ALLOWED`
* `BLOCKED_NOT_ACCEPTED_MATCH_RESULT`
* `SKIPPED_UNAVAILABLE`
* `SKIPPED_ERROR`
* `BLOCKED_UNAVAILABLE`
* `BLOCKED_ERROR`
* `BLOCKED_UNEXPECTED_STATUS`
