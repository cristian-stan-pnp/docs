title: 'Integrare Marketplace'
description: 'Cum să integrezi Paylands într-un marketplace cu mai mulți comercianți'
---

Acesta este un exemplu al modului în care ar arăta cererea finală:

```bash
curl --request POST 'https://api.paylands.com/v1/sandbox/payment' \
--header 'Authorization: pk_test_3c140607778e1217f56ccb8b50540e00' \
--header 'Content-Type: application/json' \
--data-raw '{
  "signature": "341f7de8e6fc49da8d8736473af6b03a",
  "amount": 10000,
  "operative": "AUTHORIZATION",
  "secure": false,
  "customer_ext_id": "test",
  "service": "9A1BDCC8-DB30-4ED2-8523-62B330A67873",
  "description": "Payment #1",
  "additional": null,
  "url_post": "https://my.website.com/process",
  "url_ok": "https://my.website.com/ok",
  "url_ko": "https://my.website.com/ko",
  "template_uuid": "6412549E-933E-4DFE-A225-2E87FBF7623E",
  "dcc_template_uuid": "BF418CA6-7043-4864-B36F-F02C2CF2B76B",
  "source_uuid": null,
  "save_card": true,
  "reference": "50620",
  "dynamic_descriptor": "PNP*NombreComercio",
  "expires_in": 3600,
  "extra_data": {
    "marketplace": [
      {
        "merchant_location_id": 123,
        "amount": 0,
        "description": "MarketPlace1",
        "net_amount": 150
      },
      {
        "merchant_location_id": 4487,
        "amount": 10000,
        "description": "Comercio1",
        "net_amount": 9850
      }
    ]
  }
}'
```

A fost adăugat un câmp nou la cerere numit `extra_data`. Aici trebuie să adăugați informațiile pertinente pentru ca plata să fie procesată corect atât pentru marketplace, cât și pentru comercianții implicați.

Valorile `merchant_location_id` sunt ID-urile pe care Paylands le va furniza atât pentru marketplace, cât și pentru comercianți odată ce sunteți înregistrat în sistemul nostru de marketplace-uri.

În acest exemplu, este vorba de o tranzacție de 100€ (suma este întotdeauna indicată în cenți, de aceea ar fi 10000). Suma brută de 100€ corespunde comerciantului1 cu ID-ul 4487, din care va reține 98,50€, iar restul de 1,50€ va merge către MarketPlace ca și comision.

Astfel se indică pentru fiecare comerciant ce parte din suma totală îi corespunde. Și din această parte, cu net\_amount se poate ști cât trebuie decontat către MarketPlace și către comerciant.

### Rambursări

```bash
curl --request POST 'https://api.paylands.com/v1/sandbox/payment/refund' \
--header 'Authorization: pk_test_3c140607778e1217f56ccb8b50540e00' \
--header 'Content-Type: application/json' \
--data-raw '{
  "signature": "121149a0ba5361191d740fa898784a8b",
  "order_uuid": "1f405ea3-9798-42a6-9e87-bd347ef67f55",
  "amount": 5000,
  "extra_data": {
    "marketplace_refunds": [
      {
        "merchant_location_id": 123,
        "amount": 0,
        "description": "MarketPlace1",
        "net_amount": 10
      },
      {
        "merchant_location_id": 4487,
        "amount": 5000,
        "description": "Comercio1",
        "net_amount": 4990
      }
    ]
  }
}'
```

La efectuarea unei [rambursări](/reference/#operation/refund-payment), trebuie să adăugăm câmpul `extra_data` și în interiorul acestuia `marketplace_refunds`. Va fi necesar să indicăm `amount` și `marketplace_refunds` doar în cazul rambursărilor parțiale. Acest element nou urmează același comportament descris anterior.
