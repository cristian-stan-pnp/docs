## Integración de PayRetailers[​](#integración-de-payretailers "Enlace directo al Integración de PayRetailers")

PayRetailers permite procesar **pay-ins** mediante métodos de pago locales en LATAM (PSE, EFECTY, Pago Fácil, Khipu, SPEI, PIX, Transfer y Tarjeta Colombia), así como **pay-outs**. Los pagos pueden realizarse mediante **checkout** o **redirección**.

***

## Payins con Payretailers[​](#payins-con-payretailers "Enlace directo al Payins con Payretailers")

### Requisitos generales[​](#requisitos-generales "Enlace directo al Requisitos generales")

#### Monedas soportadas[​](#monedas-soportadas "Enlace directo al Monedas soportadas")

Actualmente PayRetailers solo soporta las siguientes divisas:

* USD
* ARS
* COP
* PEN
* MEX
* BRL

#### Datos obligatorios (`extra_data`)[​](#datos-obligatorios-extra_data "Enlace directo al datos-obligatorios-extra_data")

Todos los métodos de pago requieren incluir la siguiente información dentro de `extra_data`:

* **profile**
* `first_name`
* `last_name`
* `email`
* `document_identification_number`
* `phone.number`, `phone.prefix` (si aplica)

* **billing\_address**
* `city`, `country`,`zip_code`, `address1`, `state_code`

Ejemplo:

```
"extra_data": {    "checkout": {        "uuid": "C42043F9-B77C-4918-BD43-86C526B4F24D"    },    "profile": {        "first_name": "John",        "last_name": "Doe",        "email": "johndoe@gmail.com",        "document_identification_number": "92309089",        "phone": {            "number": "02614235138",            "prefix": "54"        }    },    "billing_address": {        "city": "Bogota",        "country": "COL",        "address1": "Calle Mayor, 23",        "zip_code": "111611",        "state_code": ""    }}
```

#### Parámetros adicionales en la URL[​](#parámetros-adicionales-en-la-url "Enlace directo al Parámetros adicionales en la URL")

| Parámetro | Descripción                  | Ejemplos                                |
| --------- | ---------------------------- | --------------------------------------- |
| `apm`     | Selección del método de pago | `?apm=PSE`, `?apm=PIX`, `?apm=TRANSFER` |
| `lang`    | Idioma del flujo             | `?lang=es`, `?lang=en`, `?lang=pt`      |

Idioma por defecto: `es` (español).

#### Comportamiento del entorno sandbox[​](#comportamiento-del-entorno-sandbox "Enlace directo al Comportamiento del entorno sandbox")

El entorno **sandbox** de PayRetailers puede no reflejar completamente el comportamiento real de los métodos de pago.

Para simular distintos estados finales en sandbox, se pueden utilizar los siguientes valores de `amount`:

| `amount`             | Estado              | Detalle          |
| -------------------- | ------------------- | ---------------- |
| `11111`              | Pagado              | Pago aceptado    |
| `44444`              | Rechazado           | Pago cancelado   |
| `55555`              | Rechazado           | Pago expirado    |
| `66666`              | Rechazado           | Error en el pago |
| Cualquier otro valor | Esperando respuesta | Pago pendiente   |

## Payouts con Payretailers[​](#payouts-con-payretailers "Enlace directo al Payouts con Payretailers")

Payretailers permite realizar **payouts** (envíos de fondos) en los siguientes países:

| País     | Moneda |
| -------- | ------ |
| Brasil   | BRL    |
| México   | MXN    |
| Chile    | CLP    |
| Colombia | COP    |
| Perú     | PEN    |
| Ecuador  | USD    |

Existen dos métodos de payout:

| Método   | Descripción                                | Países compatibles   |
| -------- | ------------------------------------------ | -------------------- |
| Transfer | Transferencia bancaria tradicional         | Todos los anteriores |
| PIX      | Transferencia inmediata mediante clave PIX | Solo Brasil          |

***

### Campos obligatorios en `extra_data`[​](#campos-obligatorios-en-extra_data "Enlace directo al campos-obligatorios-en-extra_data")

#### `profile` (siempre requerido):[​](#profile-siempre-requerido "Enlace directo al profile-siempre-requerido")

* `first_name`
* `last_name`
* `email`
* `document_identification_number`
* `document_identification_type` *(opcional; solo para COL, ECU y PER. Valores posibles: `NATIONAL_IDENTITY_DOCUMENT`, `FOREIGN_IDENTIFICATION_DOCUMENT`, `FISCAL_IDENTIFICATION_CODE`, `VALID_PASSPORT`)*
* `phone.number` *(requerido solo si `country` es BRA, PER o ECU)*

#### `billing_address`:[​](#billing_address "Enlace directo al billing_address")

* `address1`
* `country`

#### `payment` (según el tipo de payout):[​](#payment-según-el-tipo-de-payout "Enlace directo al payment-según-el-tipo-de-payout")

**Si el tipo de payout es `Transfer`:**

* `bank_name` *(excepto Argentina)* – Este campo debe contener el **código** del banco del destinatario. Consulta la sección [Códigos de bancos por país](#c%C3%B3digos-de-bancos-por-pa%C3%ADs) para ver los valores válidos según el país.
* `account` [Ver especificaciones](#especificaciones-para-account-por-pa%C3%ADs-transfer)
* `account_type` *(excepto Argentina y México)*
* `account_agency_number` *(solo Brasil, obligatorio)* – Código numérico de la agencia bancaria, proporcionado por el banco. **Usar solo los números antes del guion** (ej: `"1234-5"` → enviar `"1234"`).
* `aba_swift` *(opcional y solo Brasil)*
* `payout_beneficiary_type_code` *(opcional, solo Chile. Valores: `PERSON` (por defecto), `COMPANY`)*

**Si el tipo de payout es `PIX` (solo Brasil):**

* `recipient_pix_key` *(opcional. Si no se indica, se usa el CPF como clave PIX)*

***

| ECU  | `VALID_PASSPORT`                     | Pasaporte                     |
| PER  | `NATIONAL_IDENTITY_DOCUMENT`         | DNI                           |

### Formato de clave PIX (`recipient_pix_key`)[​](#formato-de-clave-pix-recipient_pix_key "Enlace directo al formato-de-clave-pix-recipient_pix_key")

| Tipo clave | Formato regex                                                  | Ejemplo                                |
| ---------- | -------------------------------------------------------------- | -------------------------------------- |
| CPF        | `^[0-9]{11}$`                                                  | `12345678901`                          |
| Teléfono   | `^+[1-9][0-9]\d{1,14}$`                                        | `+5510998765432`                       |
| Email      | `^[a-z0-9.!#$&'*+/=?^_{}~-]+@[a-z0-9.-]+$`                     | `pix@bcb.gov.br`                       |
| Aleatorio  | `[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}` | `123e4567-e89b-12d3-a456-426655440000` |

> Nota: Si no se indica explícitamente el campo `recipient_pix_key` dentro de `extra_data.payment`, se utilizará automáticamente el valor del campo `document_identification_number` como clave PIX, siempre que este represente un CPF válido, simplificando la integración en los casos más comunes.

En el entorno de sandbox, puedes simular distintos resultados en las operaciones de payout utilizando el valor del campo `amount` que indicarás al [**generar la orden de pago**](/reference/#operation/generate-payment-order):

* **Cualquier valor distinto de `666`** simula un **payout exitoso**.
* El valor **`666`** simula un **payout rechazado**.
