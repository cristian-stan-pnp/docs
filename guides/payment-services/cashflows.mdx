## Métodos de pago en Sandbox

Los pagos exitosos en Sandbox deben tener un importe de al menos 3000€, lo que supone un importe de 300000.

La API soporta devolver códigos de error concretos iguales al importe de la operación cuando se utiliza la siguiente tarjeta: `5555555500024444 12/34 987`. Puede encontrarse más información [aquí](https://docs.gateway.clearhaus.com/#test-card-numbers).

| Resultado | Tarjeta                    |
| --------- | -------------------------- |
| OK        | 4111111111111111 12/34 123 |
| KO        | 4200000000000000 12/34 123 |

## Códigos de error

Los códigos de error en Cashflows siguen la siguiente estructura:

```
Categoría del error x Código identificativo
```

Por ejemplo:

```
Vx01
```

La letra indica la categoría del error:

* `V` significa error de validación.
* `D` y `R` significan rechazo.
* `B` significa bloqueo.
* `C` significa que el propio usuario ha cancelado el pago.
* `S` significa error del sistema.

El valor de `x` es un dígito de uso interno de Cashflows que se reemplaza por un número entre 0 y 9. Su valor no cambia el significado del error.

El mensaje de error completo tiene la siguiente forma:

```
Mensaje Código
```

Por ejemplo:

```
Non-specific decline [D101]
```
