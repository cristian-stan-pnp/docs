# SPEI

## Integração por redirecionamento[​](#integração-por-redirecionamento "Link direto para Integração por redirecionamento")

Para integrá-lo por redirecionamento, o utilizador deve estar presente no momento do pagamento, que deve ser processado através de um navegador. Em primeiro lugar, devemos [gerar uma ordem de pagamento](/reference/#operation/generate-payment-order). No caso de usar SPEI através do FLOW, precisaremos enviar obrigatoriamente o email no `extra_data`:

```
  "extra_data": {    "profile": {      "first_name": " ",      "last_name": " ",      "email": "test@gmail.com"    }  }
```

Como identificador de serviço, deve indicar o **service\_uuid** do serviço FACILITAPAY ou FLOW que tenha ativado no seu **Backend da Paylands → Administrar → Serviços de pagamento → [Lista](https://backend.paylands.com/payment-services/list)**.

Uma vez gerada a ordem, o utilizador deve ser redirecionado para [o ecrã de pagamento](/reference/#operation/redirection-payment) indicando o token da ordem e na query, o nome do APM:

Construímos o URL de redirecionamento substituindo o **token** pelo valor devolvido na chamada anterior.

```
https://api.paylands.com/v1/sandbox/payment/process/{token}?apm=SPEI
```

Uma vez que o utilizador seja redirecionado para este ecrã, poderá visualizar uma CLABE ou um código QR com o qual, através da aplicação móvel, completar o pagamento. Quando completar o pagamento, será redirecionado de volta para o URL OK ou KO, dependendo do resultado do pagamento.

## Campos personalizados[​](#campos-personalizados "Link direto para Campos personalizados")

Opcionalmente, se estiver a usar SPEI com FLOW, dentro do objeto `extra_data` pode incluir o campo `payment_reason`, que permite indicar uma descrição do pagamento.

```
  "extra_data": {    "payment": {      "payment_reason": "Payment description"    }  }
```
