# Webpay

## Integração por redirecionamento com PAYRETAILERS[​](#integração-por-redirecionamento-com-payretailers "Link direto para Integração por redirecionamento com PAYRETAILERS")

Este método requer que o utilizador esteja presente no momento do pagamento, que deve ser processado através de um navegador. Em primeiro lugar, devemos [gerar uma ordem de pagamento](/reference/#operation/generate-payment-order) indicando os seguintes campos sobre o utilizador para evitar que tenha de os preencher num passo posterior.

```
  "extra_data": {      "profile": {          "first_name": "John",          "last_name": "Doe",          "email": "johndoe@gmail.com",          "document_identification_number": "92309089",          "phone": {              "number": "02614235138",              "prefix": "54"          }      },      "billing_address": {          "city": "Castellon",          "country": "CHL",          "address1": "Avinguda del Mar 23",          "zip_code": "12000",          "state_code": "CT"      }  }
```

## Integração por redirecionamento com FLOW[​](#integração-por-redirecionamento-com-flow "Link direto para Integração por redirecionamento com FLOW")

Para integrá-lo por redirecionamento, o utilizador deve estar presente no momento do pagamento, que deve ser processado através de um navegador. Em primeiro lugar, devemos [gerar uma ordem de pagamento](/reference/#operation/generate-payment-order), indicando o email.

```
  "extra_data": {    "profile": {      "first_name": " ",      "last_name": " ",      "email": "test@gmail.com"    }  }
```

Como identificador de serviço, deve indicar o **service\_uuid** do serviço PAYRETAILERS ou FLOW que tenha ativado no seu **Backend da Paylands → Administrar → Serviços de pagamento → [Lista](https://backend.paylands.com/payment-services/list)**.

Uma vez gerada a ordem, o utilizador deve ser redirecionado para [o ecrã de pagamento](/reference/#operation/redirection-payment) indicando o token da ordem e na query, o nome do APM:

Construímos o URL de redirecionamento substituindo o **token** pelo valor devolvido na chamada anterior.

```
https://api.paylands.com/v1/sandbox/payment/process/{token}?apm=WEBPAY
```

Uma vez que o utilizador seja redirecionado para este ecrã, poderá introduzir os dados do cartão para completar o pagamento. Quando completar o pagamento, será redirecionado de volta para o URL OK ou KO, dependendo do resultado do pagamento.

## Testes[​](#testes "Link direto para Testes")

Para mais informações sobre como testar este método de pagamento em cada um dos seus fornecedores, visite a documentação da PAYRETAILERS ou FLOW.

## Campos personalizados[​](#campos-personalizados "Link direto para Campos personalizados")

Opcionalmente, no caso de estar a utilizar WEBPAY com FLOW, dentro do objeto `extra_data` pode incluir o campo `payment_reason`, que permite indicar uma descrição do pagamento.

```
  "extra_data": {    "payment": {      "payment_reason": "Payment description"    }  }
```
