# Khipu

## Integração por redirecionamento para FLOW[​](#integração-por-redirecionamento-para-flow "Link direto para Integração por redirecionamento para FLOW")

Para integrá-lo por redirecionamento, o utilizador deve estar presente no momento do pagamento, que deve ser processado através de um navegador. Em primeiro lugar, devemos [gerar uma ordem de pagamento](/reference/#operation/generate-payment-order), indicando o email.

```
  "extra_data": {    "profile": {      "first_name": " ",      "last_name": " ",      "email": "test@gmail.com"    }  }
```

Como identificador de serviço, deve indicar o **service\_uuid** do serviço FLOW que tenha ativado no seu **Backend da Paylands → Administrar → Serviços de pagamento → [Lista](https://backend.paylands.com/payment-services/list)**.

Uma vez gerada a ordem, o utilizador deve ser redirecionado para [o ecrã de pagamento](/reference/#operation/redirection-payment) indicando o token da ordem e na query, o nome do APM:

Construímos o URL de redirecionamento substituindo o **token** pelo valor devolvido na chamada anterior.

```
https://api.paylands.com/v1/sandbox/payment/process/{token}?apm=KHIPU
```

Uma vez que o utilizador seja redirecionado para este ecrã, poderá selecionar o seu banco para autorizar a transação. Quando completar o pagamento, será redirecionado de volta para o URL OK ou KO, dependendo do resultado do pagamento.

## Integração por redirecionamento para Payretailers[​](#integração-por-redirecionamento-para-payretailers "Link direto para Integração por redirecionamento para Payretailers")

Para integrá-lo por redirecionamento, o utilizador deve estar presente no momento do pagamento, que deve ser processado através de um navegador. Em primeiro lugar, devemos [gerar uma ordem de pagamento](/reference/#operation/generate-payment-order), indicando first\_name, last\_name, email, document\_identification\_number, phone, city, country, address1, zip\_code, state\_code.

```
  "extra_data": {    "checkout": {        "uuid": "C42043F9-B77C-4918-BD43-86C526B4F24D"    },    "profile": {        "first_name": "Kobe",        "last_name": "Bryant",        "email": "andreu.sanchez@paynopain.com",        "document_identification_number": "92309089",        "phone": {            "number": "02614235138",            "prefix": "54"        }    },    "billing_address": {        "city": "Castellon",        "country": "COL",        "address1": "Avinguda del Mar 23",        "zip_code": "12000",        "state_code": "CT"    }  }
```

Como identificador de serviço, deve indicar o **service\_uuid** do serviço Payretailers que tenha ativado no seu **Backend da Paylands → Administrar → Serviços de pagamento → [Lista](https://backend.paylands.com/payment-services/list)**.

Uma vez gerada a ordem, o utilizador deve ser redirecionado para [o ecrã de pagamento](/reference/#operation/redirection-payment) indicando o token da ordem e na query, o nome do APM:

Construímos o URL de redirecionamento substituindo o **token** pelo valor devolvido na chamada anterior.

```
https://api.paylands.com/v1/sandbox/payment/process/{token}?apm=KHIPU
```

## Campos personalizados[​](#campos-personalizados "Link direto para Campos personalizados")

Opcionalmente, dentro do objeto `extra_data` pode incluir o campo `payment_reason`, que permite indicar uma descrição do pagamento.

```
  "extra_data": {    "payment": {      "payment_reason": "Payment description"    }  }
```
