## Anti Fraude

A Paylands oferece um potente sistema antifraude que permite tanto redirecionar transações suspeitas para o 3DS como rejeitá-las diretamente, sem que o pagamento chegue a ser processado.

Para isso, existe uma série de verificações disponíveis que podem ser agrupadas para formar regras antifraude, cada uma com a sua própria pontuação. A soma das pontuações de todas as regras que foram ativadas será o que determinará a pontuação para a operação em curso.

### Configuração e criação do Anti Fraude

Em primeiro lugar, temos de nos dirigir ao separador Segurança -> Anti Fraude no painel. Aqui veremos uma listagem com todos os anti fraudes disponíveis e o número de serviços associados ao mesmo.

Na coluna `Servicios Totales` podemos consultar clicando no número de serviços associados ao anti fraude. Lembre-se de que tem de associar ao anti fraude os serviços que irão fazer uso do mesmo. Um serviço só pode estar associado a um antifraude.

![Anti fraud list](/images/docs/assets/images/antifraud_list-28332c37709f7d7da5e1caf87c100030.png)

Para atribuir um serviço devemos dirigir-nos a Administrar -> Listado de Servicios -> Listado. Acedemos ao detalhe do serviço que desejamos associar a um Anti Fraude. No separador Anti Fraude escolhemos o desejado e atualizamos.

![Anti fraud service](/images/docs/assets/images/associate_antifraud_service-af03a69630483f7eaa9c50e4f4e49680.png)

## Listas de segurança

Dentro das listas de segurança podemos configurar os elementos (IP, BIN, UUID, EXTERNAL\_ID e TOKEN) para que sejam marcados como seguros, de risco ou bloqueados.

Para aceder a cada uma das listas de segurança basta dirigirmo-nos a **Segurança** e clicar na lista desejada.

Nesta secção poderemos ver cada uma das listas disponíveis onde poderemos ativá-las ou desativá-las e consultar os serviços associados às mesmas. Podendo inclusivamente desvincular os serviços que pretendamos dessa lista.

![ColorList table](/images/docs/assets/images/colorlist_table-ec48062c05b366a4785978833e314c1e.png)

![ColorList services](/images/docs/assets/images/colorlist_services-5d2d074e0b68dd664ee3e6f76f2608db.png)

No detalhe de cada lista podemos criar, modificar e eliminar elementos. Lembre-se de que também terá de associar tantos serviços quantos desejar que utilizem esta lista.

![ColorList add service](/images/docs/assets/images/colorlist_add_service-77491fc9efcb98ef490ee2cae1f8299f.png)

### Listas Brancas

A partir desta secção pode configurar listas brancas que podem ser associadas aos serviços existentes.

Estas listas permitem configurar verificações para evitar que uma operação de confiança entre em avaliação de risco ou fraude. Os parâmetros que podem ser verificados para validar que a operação é de confiança são os seguintes:

* IP: Endereço IP a partir do qual a operação é realizada.
* BIN: Primeiros 6 dígitos da numeração do cartão.
* UUID: Identificador do cartão.
* ID\_EXTERNO: Identificador do customer.
* TOKEN: Token identificador do cartão.

### Listas Negras

A partir desta secção pode configurar listas negras que podem ser associadas aos serviços existentes.

Estas listas permitem configurar verificações para evitar que uma operação de confiança duvidosa seja processada. Os parâmetros que podem ser verificados para validar que a operação deve ser bloqueada são os seguintes:

* IP: Endereço IP a partir do qual a operação é realizada.
* BIN: Primeiros 6 dígitos da numeração do cartão.
* UUID: Identificador do cartão.
* ID\_EXTERNO: Identificador do customer.
* TOKEN: Token identificador do cartão.

### Listas Cinzentas

A partir desta secção pode configurar listas cinzentas que podem ser associadas aos serviços existentes.

Estas listas permitem configurar verificações para obrigar uma operação de confiança duvidosa a ser processada como uma ordem segura (3DS). Os parâmetros que podem ser verificados para validar que o nível de segurança da operação deve ser aumentado são os seguintes:

* IP: Endereço IP a partir do qual a operação é realizada.
* BIN: Primeiros 6 dígitos da numeração do cartão.
* UUID: Identificador do cartão.
* ID\_EXTERNO: Identificador do customer.
* TOKEN: Token identificador do cartão.
