São suportados dois métodos de pagamento: Cartão e [Bizum](/docs/apm/apm-bizum). As operações de devolução, confirmação, cancelamento e recuperação de estado são compatíveis com ambos os métodos de pagamento. As demais operações são orientadas apenas a cartões.

## Restrições e requisitos

O Redsys suporta pelo menos as seguintes moedas: AUD, BGN, CAD, CHF, COP, CZK, DKK, EUR, GBP, HRK, HUF, ILS, INR, JPY, MYR, NOK, NZD, PEN, PLN, RUB, SAR, SEK, SGD, THB, USD, ZAR.

### Extra data

Não é necessário indicar `extra_data` ao criar a ordem para realizar os pagamentos.

### Dynamic descriptor

O campo `dynamic_descriptor` é opcional e, caso seja indicado, só poderá conter no máximo 38 caracteres e permite números, letras e espaços em branco.

### Reference

O campo `reference` é opcional. Caso seja indicado, este valor aparecerá em **Canales** do Redsys dentro do campo `descripcion`, junto com o identificador `uuid` da transação.

O formato exibido em Canales será:

```
uuid_transaccion/reference
```

⚠️ Importante: o comerciante **não deve incluir** o `uuid` nem a barra `/` no valor de `reference`. Estes são adicionados automaticamente. O `uuid` corresponde à transação gerada no nosso sistema.

Por exemplo, se o comerciante enviar como `reference`:

```
1234
```

Em Canales será exibido:

```
10FF94B5-FE7C-41A0-B956-BBBBAA6BCA77/1234
```

Este campo permite que os comerciantes adicionem informação própria para facilitar a rastreabilidade das operações. O tamanho máximo deste campo é **88 caracteres**. Se este limite for ultrapassado, o valor de `reference` será **truncado** em Canales.

