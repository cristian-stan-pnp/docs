## Integração do PayRetailers

O PayRetailers permite processar **pay-ins** através de métodos de pagamento locais na LATAM (PSE, EFECTY, Pago Fácil, Khipu, SPEI, PIX, Transfer e Tarjeta Colombia), bem como **pay-outs**. Os pagamentos podem ser realizados mediante **checkout** ou **redirecionamento**.

***

## Payins com Payretailers

### Requisitos gerais

#### Moedas suportadas

Atualmente o PayRetailers apenas suporta as seguintes divisas:

* USD
* ARS
* COP
* PEN
* MXN
* BRL

#### Dados obrigatórios (`extra_data`)

Todos os métodos de pagamento requerem a inclusão da seguinte informação dentro de `extra_data`:

* **profile**
* `first_name`
* `last_name`
* `email`
* `document_identification_number`
* `phone.number`, `phone.prefix` (se aplicável)

* **billing\_address**
* `city`, `country`, `zip_code`, `address1`, `state_code`

Exemplo:

```
"extra_data": {    "checkout": {        "uuid": "C42043F9-B77C-4918-BD43-86C526B4F24D"    },    "profile": {        "first_name": "John",        "last_name": "Doe",        "email": "johndoe@gmail.com",        "document_identification_number": "92309089",        "phone": {            "number": "02614235138",            "prefix": "54"        }    },    "billing_address": {        "city": "Bogota",        "country": "COL",        "address1": "Calle Mayor, 23",        "zip_code": "111611",        "state_code": ""    }}
```

#### Parâmetros adicionais no URL

| Parâmetro | Descrição                       | Exemplos                                |
| --------- | ------------------------------- | --------------------------------------- |
| `apm`     | Seleção do método de pagamento  | `?apm=PSE`, `?apm=PIX`, `?apm=TRANSFER` |
| `lang`    | Idioma do fluxo                 | `?lang=es`, `?lang=en`, `?lang=pt`      |

Idioma por defeito: `es` (espanhol).

#### Comportamento do ambiente sandbox

O ambiente **sandbox** do PayRetailers pode não refletir completamente o comportamento real dos métodos de pagamento.

Para simular diferentes estados finais em sandbox, podem ser utilizados os seguintes valores de `amount`:

| `amount`             | Estado              | Detalhe            |
| -------------------- | ------------------- | ------------------ |
| `11111`              | Pago                | Pagamento aceite   |
| `44444`              | Rejeitado           | Pagamento cancelado|
| `55555`              | Rejeitado           | Pagamento expirado |
| `66666`              | Rejeitado           | Erro no pagamento  |
| Qualquer outro valor | A aguardar resposta | Pagamento pendente |

### Métodos de pagamento

#### PSE (Colômbia)

* **País:** `COL`
* **Moedas:** COP, USD
* **APM:** PSE

```
"extra_data": {    "profile": {        "first_name": "John",        "last_name": "Doe",        "email": "test@example.com",        "document_identification_number": "92309089",        "phone": { "number": "02614235138", "prefix": "54" }    },    "billing_address": {        "city": "Bogota",        "country": "COL",        "address1": "Calle 12",        "zip_code": "111611",        "state_code": ""    }}
```

## Payouts com Payretailers

O Payretailers permite realizar **payouts** (envios de fundos) nos seguintes países:

| País     | Moeda |
| -------- | ----- |
| Brasil   | BRL   |
| México   | MXN   |
| Chile    | CLP   |
| Colômbia | COP   |
| Peru     | PEN   |
| Equador  | USD   |

Existem dois métodos de payout:

| Método   | Descrição                                  | Países compatíveis   |
| -------- | ------------------------------------------ | -------------------- |
| Transfer | Transferência bancária tradicional         | Todos os anteriores  |
| PIX      | Transferência imediata mediante chave PIX  | Apenas Brasil        |

***

### Campos obrigatórios em `extra_data`

#### `profile` (sempre requerido):

* `first_name`
* `last_name`
* `email`
* `document_identification_number`
* `document_identification_type` *(opcional; apenas para COL, ECU e PER. Valores possíveis: `NATIONAL_IDENTITY_DOCUMENT`, `FOREIGN_IDENTIFICATION_DOCUMENT`, `FISCAL_IDENTIFICATION_CODE`, `VALID_PASSPORT`)*
* `phone.number` *(requerido apenas se `country` for BRA, PER ou ECU)*

#### `billing_address`:

* `address1`
* `country`

#### `payment` (conforme o tipo de payout):

**Se o tipo de payout for `Transfer`:**

* `bank_name` *(exceto Argentina)* – Este campo deve conter o **código** do banco do destinatário. Consulte a secção [Códigos de bancos por país](#c%C3%B3digos-de-bancos-por-pa%C3%ADs) para ver os valores válidos conforme o país.
* `account` [Ver especificações](#especifica%C3%A7%C3%B5es-para-account-por-pa%C3%ADs-transfer)
* `account_type` *(exceto Argentina e México)*
* `account_agency_number` *(apenas Brasil, obrigatório)* – Código numérico da agência bancária, fornecido pelo banco. **Usar apenas os números antes do hífen** (ex: `"1234-5"` → enviar `"1234"`).
* `aba_swift` *(opcional e apenas Brasil)*
* `payout_beneficiary_type_code` *(opcional, apenas Chile. Valores: `PERSON` (por defeito), `COMPANY`)*

**Se o tipo de payout for `PIX` (apenas Brasil):**

* `recipient_pix_key` *(opcional. Se não for indicado, utiliza-se o CPF como chave PIX)*

***

### Valores de `document_identification_type` por país

O campo `document_identification_type` é incluído dentro de `profile`, é **opcional** e **apenas pode ser utilizado para os países Colômbia, Equador e Peru**. Se não for indicado, será utilizado por defeito `NATIONAL_IDENTITY_DOCUMENT`. Para os restantes países, este campo não deve ser enviado.

| País | Valor `document_identification_type` | Tipo de documento equivalente |
| ---- | ------------------------------------ | ----------------------------- |
| ARG  |                                      | DNI                           |
| BR   |                                      | CPF                           |
| CL   |                                      | RUT                           |
| MX   |                                      | CURP                          |
| COL  | `NATIONAL_IDENTITY_DOCUMENT`         | Cédula de cidadania           |
| COL  | `FOREIGN_IDENTIFICATION_DOCUMENT`    | Documento de estrangeiro      |
| ECU  | `NATIONAL_IDENTITY_DOCUMENT`         | DNI                           |
| ECU  | `FISCAL_IDENTIFICATION_CODE`         | RUC                           |
| ECU  | `VALID_PASSPORT`                     | Passaporte                    |
| PER  | `NATIONAL_IDENTITY_DOCUMENT`         | DNI                           |
| PER  | `FOREIGN_IDENTIFICATION_DOCUMENT`    | Cartão de estrangeiro         |
| PER  | `FISCAL_IDENTIFICATION_CODE`         | RUC                           |
| PER  | `VALID_PASSPORT`                     | Passaporte                    |

***

| account\_type  | BR | CO | CL | PE | EC | MX |
| -------------- | -- | -- | -- | -- | -- | -- |
| CHECKING       | ✓  | ✓  | ✓  | ✓  | ✓  |    |

| País           | Comprimento esperado | Comentários adicionais                                                                      |
| -------------- | -------------------- | ------------------------------------------------------------------------------------------- |
| AR (Argentina) | 22                   | CBU ou CVU                                                                                  |
| BR (Brasil)    | 13                   | —                                                                                           |
| CL (Chile)     | 16                   | —                                                                                           |
| CO (Colômbia)  | 16                   | —                                                                                           |
| MX (México)    | 16, 18               | General (ACCOUNT\_CLABE): 18 dígitos Número de cartão de débito (CARD\_NUMBER): 16 dígitos  |
| EC (Equador)   | 12                   | —                                                                                           |
| PE (Peru)      | 10, 13, 14, 18, 20   | Depende do banco. [Ver detalhe](#detalhe-para-peru-account)                                 |

##### Detalhe para Peru (`account`)

| Banco / Instituição      | Comprimento esperado | Comentários / Exemplos                                   |
| ------------------------ | -------------------- | -------------------------------------------------------- |

| Scotiabank               | 10                   | 3 dígitos agência + 7 conta. Ex.: `0037651234`           |
| Outros bancos / IFIs     | 20                   | CCI obrigatório. Ex.: `00219400254640654321`             |

### Formato de chave PIX (`recipient_pix_key`)

| Tipo de chave | Formato regex                                                  | Exemplo                                |
| ------------- | -------------------------------------------------------------- | -------------------------------------- |
| CPF           | `^[0-9]{11}$`                                                  | `12345678901`                          |
| Telefone      | `^\+[1-9][0-9]\d{1,14}$`                                        | `+5510998765432`                       |
| Email         | `^[a-z0-9.!#$&'*+/=?^_{}~-]+@[a-z0-9.-]+$`                     | `pix@bcb.gov.br`                       |
| Aleatório     | `[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}` | `123e4567-e89b-12d3-a456-426655440000` |

> Nota: Se não for indicado explicitamente o campo `recipient_pix_key` dentro de `extra_data.payment`, será utilizado automaticamente o valor do campo `document_identification_number` como chave PIX, desde que este represente um CPF válido, simplificando a integração nos casos mais comuns.

### Testes no ambiente de sandbox

No ambiente de sandbox, pode simular diferentes resultados nas operações de payout utilizando o valor do campo `amount` que indicará ao [**gerar a ordem de pagamento**](/reference/#operation/generate-payment-order):

* **Qualquer valor diferente de `666`** simula um **payout bem-sucedido**.
* O valor **`666`** simula um **payout rejeitado**.
